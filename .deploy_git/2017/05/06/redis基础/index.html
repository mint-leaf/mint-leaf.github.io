redis-python 的一些基础操作

  • redis中的字符串操作指令：

    • get ： get(key-name)

      • 获得key-name的值

    • set ： set(key-name)

      • 设置key-name的值

    • incr： incr(key-name, num)

      • 实现key-name的自增操作，默认从0开始计数，
      • 如果不指定num，默认+1(s)
      • 可以指定num的值来实现加的步长值

    • decr : decr(key-name, num)

      • 实现key-name的自减操作， 默认从0开始计数，
      • 其他同incr

    • append : append(key-name, value)

      • 将value加到key-name -> value的末尾
      • 如果key-name这个键不存在，则新建这个键，值为””
      • return len(key-name -> value)

    • getrange : getrange(key-name, start?int, end?int)

      • 获得key-name -> value[start, end]的子串
      • 如果end>length，那么只会返回[start, len(value)-1]
      • 如果end<0，那么会返回key-name -> value[start, len(value)+end]

    • setrange : setrange(key-name, offset?int, value1)

      • 将从key-name -> value[offset-1]设置成value
      • value1可以是一个字符串

    • getbit : getbit(key-name, offset?int)

      • 将字符串看成二进制，返回key-name -> value[offset]的二进制数值

    • setbit : setbit(key-name, offset?int, value?(0 or 1))

      • 将key-name -> value对应的bit-value的bit-value[offset]设置成value
      • 如果offset>len,那么就会把offset到len(bit-value)前面的位设置成0
      • 对set-bit后的键进行get后，会返回bit-value对应的字符串
      • 解释bit-value的时候是按照偏移量从高到低进行解释的

    • bitcount : bitcount(key-name, [start, end])

      • 对key-name -> bit-value进行1的数量的统计
      • 指定了start， end的话只对bit-value[start, value]的范围进行统计
      • 对于[start, end]的解释参照getrange的

    • bitop : bitop(operation?func, dest-key, [key-name, …])

      • operation?func: and, or, xor, not 位运算操作
      • 对[key-name, …]里面的key-name(s) -> bit-value(s) 进行operation对应的操作
      • 将结果保存在dest-ke

  • 关于redis中的列表操作指令：

    • rpush : rpush (key-name, *[value0, …])

      • 将一个或者多个值从右端推入 key-name -> value-list
      • 返回len

    • lpush : lpush (key-name, *[value0, …])

      • 将一个或者多个值从左端推入 key-name -> value-list
      • 返回len

    • rpop ： rpop (key-name)

      • 移除并返回key-name -> value-list的最右端元素
      • 当key-name -> value-list为空时，返回None

    • lpop : lpop (key-name)

      • 移除并返回key-name -> value-list的最左端元素

    • lindex : lindex(key-name, offset?int)

      • 返回key-name -> value-list[offset]
      • 如果offset > len(value-list), 则返回None

    • lrange : lrange(key-name, start, end)

      • 返回key-name -> value-list[start, end]的元素，[start, end]是闭区间

    • ltrim ： ltrim(key-name, start, end)

      • 切片操作， 只保留key-name -> value-list[start, end]的元素
      • [start, end]是闭区间
      • 返回True/False

    • blpop : blpop([key-name0, …], time-out):

      • 从第一个非空key-name -> value-list弹出最左端的元素
      • 在time-out秒内阻塞等待可弹出的元素出现

    • brpop : brpop([key-name0, …], time-out):

      • 从第一个非空key-name -> value-list弹出最右端的元素
      • 在time-out秒内阻塞等待可弹出的元素出现

    • rpoplpush : rpoplpush(source-key, dest-key)

      • 从source-key -> source-list 弹出位于最右端的元素并且将这个元素压入dest-key -> dest-list最左端
      • 向用户返回这个元素

    • brpoplpush : brpoplpush(source-key, dest-key, time-out)

      • rpoplpush的阻塞模型

  • 关于redis中的集合操作指令：

    • sadd : sadd(key-name, *[item, …])

      • 将一个或者多个item加入到key-name -> value-set中
      • 返回原key-name -> value-set不存在的元素的数量

    • srem : srem(key-name, *[item, …])

      • 将一个或者多个item从key-name -> value-set 中移除
      • 返回移除元素的数量
      • 如果[item, …]都不存在， 返回0

    • sismember : sismember(key-name, item)

      • 检查item是否属于key-name -> value-set
      • return boolean

    • scard : scard(key-name)

      • 返回key-name -> value-set的数量
      • 如果key-name -> value-set不存在， 返回0

    • smembers : smembers(key-name)

      • 返回key-name -> value-set中的所有元素

    • srandmember : srandmember(key-name, number=None)

      • 如果number = None, 随机从key-name -> value-set返回一个元素, type=str
      • 如果number > 0, 那么会随机返回number个元素, type=list, 不会有重复
      • 如果number = 0, 那么会随机返回1个元素, type=str
      • 如果number > 0, 那么会随机返回number个元素, type=list, 会有重复

    • spop : spop(key-name)

      • 随机从从key-name -> value-set弹出一个元素
      • 返回被移除的元素

    • smove : smove(source-key, dest-key, item)

      • 从source-key -> source-set 中弹出元素 item 并将item加入到dest-key -> dest-set中
      • 成功后返回1, 失败后返回0

    • sdiff : sdiff(key-name0, *[key-name1, …])

      • 返回只存在于key-name0中的元素
      • 数学上的差集运算

    • sdiffstore : sdiffstore(dest-key, key-name, *[key-name1, …])

      • 将sdiff的结果存储到dest-key -> dest-set中
      • 返回放入dest-set的元素数量
      • dest-set中的元素会被清空

    • sinter ： sinter(key-name0, *[key-name1, …])

      • 返回同时存在于key-name(s) -> value-set中的元素
      • 数学上的交集运算

    • sinterstore ： sinterstore(dest-key, key-name0, *[key-name1, …])

      • 将sinter的结果存储到dest-key -> dest-set中
      • 其他类似于sdiffstore

    • sunion ： sunion(key-name0, *[key-name1, …])

      • 返回至少存在于一个集合的元素
      • 数学上的并集运算

    • sunionstore ： sunionstore(dest-key, key-name0, *[key-name1, …])

      • 将sunion的结果存储到dest-key -> dest-set中
      • 其他类似于sdiffstore

  • 关于redis中的散列操作指令：

    • hmset : hmset(key-name, {key1:value1, …})

      • 为key-name -> value-hash 添加键值对
      • 返回True/False

    • hmget : hmget(key-name, [key1, …])

      • 从key-name -> value-hash 返回[key1, …]对应的[value1, …]
      • 返回的Type=list
      • 如果在[key1, …]存在key-name -> value-hash不存在的key, 那么返回None

    • hdel : hdel(key-name, [key1, …])

      • 删除key-name -> value-hash 中[key1, …]对应的[value1, …]
      • 返回成功找到并删除的key：value的数量

    • hlen : hlen(key-name)

      • 返回key-name -> value-hash中键值对的数量

    • hexists : hexist(key-name, key)

      • 检查key是否存在于key-name -> value-hash中
      • return true/false

    • hkeys : hkeys(key-name)

      • 返回key-name -> value-hash的所有键
      • type=list

    • hvalues : hvalues(key-name)

      • 返回key-name -> value-hash的所有值
      • type=list

    • hgetall : hgetall(key-name)

      • 返回key-name -> value-hash的所有键值对

    • hincrby : hincrby(key-name, key, incrment)

      • 将key-name -> value-hash[key] 加上incrment
      • 返回key -> value + incrment, type=long

    • hincrbyfloat : hincrbyfloat(key-name, key, float)

      • 将key-name -> value-hash[key] 加上float
      • 返回key -> value + float, type=float

  • 关于redis中的有序集合(zset)操作指令：

    • zadd : zadd(key-name, **{member: score?float, …})[推荐使用]

      • 将带有给定score的member加入到key-name -> value-set中
      • 返回加入到key-name -> value-zset 的数量
      • 除了**{member: score?float, ...}， 还可以使用*[member0, score0?float, ...]

    • zrem : zrem(key-name, *[member0, …])

      • 从key-name -> value-zset中移除 *[member0, …]
      • 返回被移除成员的数量

    • zcard : zcard(key-name)

      • 返回key-name -> value-zset的成员数量

    • zincrby : zincrby(key-name, member, incrment)

      • 将key-name -> value-zset 的member的分值加上incrment
      • 在redis-cli的交互模式中， 是zincrby(key-name, incrment, member)
      • 返回member -> score + incrment, type=float

    • zcount : zcount(key-name, min, max)

      • 返回分值介于min， max之间的成员数量

    • zrank : zrank(key-name, member)

      • 返回member在key-name -> value-zset的排名
      • 如果member不存在， 返回None

    • zscore : zscore(key-name, member)

      • 返回key-name -> value-zset 中member的分数
      • 如果member不存在， 返回None

    • zrange : zrange(key-name, strat, end, withscores=False)

      • 返回排名介于start和end之间的成员, type=list, element=member
      • start和end的规则参照之前的
      • 如果withscores=True， 那么element=(member, score)

    • zrevrank : zrevrank(key-name, member)

      • 返回member在key-name -> value-zset的逆序排名
      • 逆序是指score的desc顺序

    • zrevrange : zrevrange(key-name, strat, end, withscores=False)

      • zrange的逆序操作

    • zrangebyscore : zrangebyscore(key, min, max, withscores=False, score_cast_func?func)

      • 返回分值介于 min 和 max 之间的元素
      • score_cast_func?func可以用lambda表达式

    • zrevrangebyscore : zrangebyscore(key, min, max, withscores=False, score_cast_func?func)

      • zrangebyscore的逆序操作

    • zremrangebyrank : zremrangebyrank(key-name, start, end)

      • 移除key-name -> value-zset中排名介于start和end之间的元素
      • start和end是闭集
      • 返回移除的元素数量

    • zremrangebyscore : zremrangebyscore(key-name, min, max)

      • 移除key-name -> value-zset中分数介于min和max之间的元素
      • min和max是闭集
      • 返回移除元素的数量

    • zinterstore : zinterstore(dest-key, [key-name0, …], [aggregate?func])

      • 对于给定的[key-name0, …]实现类似于集合的交集运算， 结果放于dest-key中
      • aggregate?func 可选sum|min|max 函数
      • 默认使用sum运算， 将[key-name0, …]中共同member的score相加，得到dest-key -> zset-dest中member的score
      • min|max的功能以此类推

    • zunionstore : zunionstore(dest-key, [key-name0, …], [aggregate?func])

      • 对于给定的[key-name0, …]实现类似于集合的并集运算， 结果放于dest-key中
      • 其余类似于zinterstore
